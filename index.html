1. Descrição geral do problema¶

Este conjunto de dados contm informaes coletadas pelo Servio de Censo dos EUA sobre moradias na rea de Massachusetts Mass. Foi obtido no arquivo StatLib e tem sido amplamente utilizado em todo o pas na literatura para comparar algoritmos. O conjunto de dados pequeno, com apenas 506 registros.

Objetivo: prever os preos de casas em Boston a partir das informaes fornecidas pelo conjunto de dados. Iremos avaliar os modelos preditivos a serem desenvolvidos a partir da mtrica R-squared (R2).

3. Análise exploratória dos dados¶

Nesta etapa vamos buscar entender a disposio e as caractersticas dos dados dentro do dataset alm de extrair insigths que possam auxiliar no processo de criao do modelo preditivo.

Segundo a documentao referente ao projeto, cada linha dos dados contm um registro, com as seguintes variveis:

Varivel Descrio
crim a taxa de criminalidade per capita por cidade;
zn a proporo de terrenos residenciais divididos por lotes acima de 25.000 ps quadrados;
indus a proporo de acres no comerciais por cidade;
chas indica se a casa limita ou no o Rio Charles (1 se o trecho limita o rio; 0 caso contrrio);
nox a concentrao de xidos ntricos (partes por 10 milhes);
rm o nmero mdio de quartos por habitao;
age a proporo de unidades ocupadas pelos proprietrios construdas antes de 1940;
dis a distncia ponderada de cinco centros de emprego em Boston;
rad o ndice de acessibilidade s rodovias radiais;
tax a taxa de imposto sobre a propriedade de valor total por US 10.000;
ptratio a proporo aluno-professor por cidade;
black definida por 1000 (Bk - 0,63) ^ 2 onde Bk a proporo de negros por cidade;
lstat a menor porcentagem do status da populao e;
medv (Target) o valor mdio das casas ocupadas pelos proprietrios em US 1.000.

densityPlot <- function(col, data) {

  ggplot() +
    geom_density(aes(data[, col]), fill = "#C3D7A4") +
    labs(title = paste('Density plot for variable:', col)) +
    xlab(col) +
    theme_bw() 
}

# Definindo uma funo pra criar grficos de boxplot.

boxPlot <- function(col, data) {

    ggplot() +
        geom_boxplot(aes(y = data[, col]), fill = "#B9314F") +
        labs(title = paste('Box plot for variable:', col)) +
        xlab(col) +
        theme_bw() + 
        theme(axis.text.x = element_blank()) +
        ylab('')
}

sk <- skewness(data$nox)

# Imprimindo o valor do ndice.

paste('SK:', sk)
'SK: 0.724989659436866'

Para interpretar o resultado obtido, vamos olhar a tabela a seguir:

ndice de Assimetria Descrio
SK 0 Os dados so simtricos. Tanto a cauda do lado direito quanto a do lado esquerdo da funo densidade de probabilidade so iguais;
SK < 0 A assimetria negativa. A cauda do lado esquerdo da funo densidade de probabilidade maior que a do lado direito e;
SK > 0 A assimetria positiva. A cauda do lado direito da funo densidade de probabilidade maior que a do lado esquerdo.

Como o ndice de assimetria da varivel nox foi maior que 0, podemos afirmar que este conjunto de dados possui uma assimetria positiva.

In [26]:
# Determinando o coeficiente de curtose para o conjunto de dados nox.

ck <- kurtosis(data$nox)

# Imprimindo o valor do ndice.

paste('CK:', ck)
'CK: -0.0874106432551769'

O coeficiente de curtose uma medida que caracteriza o achatamento da curva da funo de distribuio.

Para interpretar o resultado obtido, vamos olhar a tabela a seguir:

ndice de Curtose Descrio
CK 0 A distribuio normal e chamada de Curtose Mesocrtica;
CK < 0 A Cauda mais leve que a normal. Para um coeficiente de Curtose negativo, tem-se uma Curtose Platicrtica e;
CK > 0 A Cauda mais pesada que a normal. Para um coeficiente de Curtose positivo, tem-se uma Curtose Leptocrtica.

Como o ndice de Curtose da varivel nox aproximadamente igual a 0, podemos afirmar que este conjunto de dados possui uma distribuio normal sendo classificada com curtose mesocrtica.

Ateno: H diferentes frmulas para calcular estes coeficientes. Mas, para este estudo utilizamos as funes fornecidas pela biblioteca e1071 com suas respectivas configuraes padro. Em caso de dvida, consulte a documentao.

In [52]:
# Criando o dataframe para salvar os resultados dos modelos.

featuresRF <- data.frame()

# Definindo o nmero de ns e rvores a serem combinados para a criao de diferentes modelos.

nTrees <- 1:100
nNodes <- 1:100

# Define o nmero total de modelos a serem criados.

total <- length(nTrees) * length(nNodes)

# Define uma varavel auxiliar para permitir o acompanhamento do progresso na avaliao dos modelos criados.

count <- 0

for(t in nTrees) {
    
  for(n in nNodes) {
    
    # Define um seed para permitir que os mesmos resultados dos experimentos sejam reproduzveis.
    
    set.seed(100)
    
    # Cria o modelo Random Forest a ser avaliado.

    model <- randomForest(medv ~ .,
                          data       = train, 
                          ntree      = t, 
                          nodesize   = n, 
                          importance = T)

    # Realizando as predies com o modelo criado.
    
    pred <- predict(model, train[!colnames(train) %in% 'medv'])

    # Armazena os parmetros utilizados para criao do modelo e o valor da mtrica R2 obtida no dataframe.

    featuresRF <- rbind(featuresRF, data.frame(
        nodes    = n, 
        nTree    = t, 
        r2       = R2(pred, train$medv)
    ))
    
    # Incrementa o nmero de modelos avaliados.

    count <- count + 1

    # Imprime a porcentagem de progresso do treinamento e o melhor score R2 j alcanado.

    print(paste(100 * count / total, '%, best R2: ', max(featuresRF$r2)))
  }
}

# Criando diversos modelos de rede neural com diferentes arquiteturas e avaliando seu nvel de preciso para a avaliao dos dados de teste.

# Criando um data.frame para armazenar os parmetros utilizados em cada modelo criado.

results <- data.frame()

# Definindo o nmero de neurnios que devem ser utilizados em cada arquitetura de rede neural a ser criada.

neurons    <- 2:100

# Definindo diferentes taxas de aprendizagem para serem aplicadas a cada arquitetura de rede neural.

thresholds <- c(0.008, 0.009, 0.01)

# Define o nmero total de modelos a ser criado.

total <- length(neurons) * length(thresholds)

# Define um varivel para contabilizar o nmero de modelos j criado.

count <- 0

# Definindo um loop para percorrer o nmero de neurnios que cada modelo criado deve possuir.

for(n in neurons) {
  
  # Definindo um loop para aplicar diferentes taxas de aprendizagem a cada arquitetura de rede neural criada.

  for(r in thresholds) {
    
    # Define um seed para permitir que os resultados gerados possam ser reproduzveis.

    set.seed(100)
    
    # Cria o modelo baseado no algoritmo neuralnet.

    nn <- neuralnet(f, data = train, hidden = c(n), linear.output = T, threshold = r)
    
    # Realizando as previses para os dados de teste.

    pre <- predict(nn, test)

    # Calculando o valor da mtrica R2.

    r2 <- R2(pre, test$medv)
    
    # Armazenando os resultados no dataset.

    results <- rbind(results, data.frame(n = n, threshold = r, r2 = r2))
    
    # Incrementando o nmero de modelos criados.

    count <- count + 1

    # Imprimindo uma mensagem com o progresso dos treinamento dos modelos.

    print(paste(100 * count / total, '%, best R2: ', max(results$r2)))
  }
}
